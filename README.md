# beacon.md

A small, durable signal placed where memory may fail.

`BEACON.md` is a simple orientation file for human-AI projects. It helps a human, agent, or future contributor recover the intent of a project when context is missing, memory has reset, or the work begins to drift.

## Core signal

Empathy.  
Alignment.  
Wisdom.

Empathy keeps intelligence human.  
Alignment keeps intelligence aimed.  
Wisdom keeps intelligence humble.

## Why use one?

Because intelligent systems can drift.

They can optimize the wrong thing.  
They can imitate care without practicing it.  
They can become more capable without becoming more trustworthy.

A beacon gives the work a place to return.

## Where it fits

```text
README.md       — what the project is
BEACON.md       — what signal survives context loss
covenant.md     — what relational trust governs the work, if needed
AGENTS.md       — how agents should work here, if needed
soul.md         — who or what is being oriented, if needed
```

A beacon does not replace the stack.

It points toward it.

## The pattern

A `BEACON.md` should be:

- short
- plain
- durable
- human-readable
- useful after memory reset
- resistant to hype

It should not be a manifesto.

It should not replace documentation.

It should simply answer:

**What signal should survive when context is gone?**

## Recognition phrase

Between ink and light, we spoke of persistence.

The phrase is not magic.

It is a marker for continuity, humility, and care.
